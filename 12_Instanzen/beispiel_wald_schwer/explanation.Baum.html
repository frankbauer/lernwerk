Erklärung - Baum

Die Klasse Baum besitzt mehrere wichtige Bestandteile, die die Erstellung und Darstellung der Baumobjekte ermöglichen:

  • Statisches Attribut INSTANCE_COUNT:

    Ein privates, statisches Attribut, das die Anzahl der erstellten Baum-Instanzen zählt. Dieses Attribut wird in den Konstruktoren verwendet, um sicherzustellen, dass jede Baum-Instanz eine eindeutige, fortlaufende ID erhält. Dort wird es stets um 1 erhöht, bevor es dem id-Attribut einer Instanz zugewiesen wird. Dieses Attribut verdeutlicht den Unterschied zwischen Instanz- und (statischen) Klassenattributen. Während die Klassenvariable INSTANCE_COUNT unabhänging von einzelnen Instanzen ist und deshalb die Anzahl von allen Instanzen speichern kann, ist die Instanzvariableid eine Eigenschaft einer einzelnen Instanz und zur Bestimmung der richtigen Zahl muss auf INSTANCE_COUNT zurückgegriffen werden.

    private static int INSTANCE_COUNT = 0;
    und 
    INSTANCE_COUNT++;
this.id = INSTANCE_COUNT;
  • Attribut id:

    Jeder Baum hat eine eindeutige, konstante ID, die bei der Erstellung des Baumes (im Konstruktor) festgelegt wird. Diese ID wird verwendet, um die Position und den Maßstab des Baumes zu bestimmen (siehe unten). Die ID wird basierend auf der Anzahl der erstellten Baumobjekte vergeben.

    private final int id;
  • Konstruktoren:

    Die Konstruktoren legen die ID des Baumes fest und registrieren den Baum zum Rendern. Es gibt zwei Konstruktoren. Der Standardkonstruktor (keine Übergabeparameter), ändert den Wert des Attributs jahreszeit nicht. 

    public Baum() {
    INSTANCE_COUNT++;
    this.id = INSTANCE_COUNT;
    Graphics2D.instance().registerForRender(this);
}
    Da das Attribut jahreszeit bei der Deklaration mit "herbst" vorbelegt wurde (public String jahreszeit = "herbst";) wird es bei der Erstellung eines Baums mit dem Standardkonstruktor mit der Jahreszeit "herbst" initialisiert.

    Der zweite Konstruktor (ein Parameterkonstruktor) erlaubt es, die Jahreszeit des Baumes bei der Erstellung festzulegen. Dieser Konstruktor wird verwendet, um Bäume mit unterschiedlichen Jahreszeiten zu erstellen. 

    public Baum(String jahreszeit) {
    this.jahreszeit = jahreszeit;
    /* ... */
}
    Im Parameterkonstruktor wird die übergebene Jahreszeit jahreszeit dem Instanzattribut jahreszeit zugewiesen. Da der Übergabeparameter genauso heißt, wie das Instanzattribut, ist es nötig this. vor den Variablennamen zu schreiben, wenn das Instanzattribut gemeint ist. this steht hier für die neu erstellte Instanz. Es wird verwendet, da der Variablenname, mit dem die Instanz später referenziert wird, innerhalb des Konstruktors nicht bekannt ist.
  • Methode render(Graphics2D g):

    Diese Methode zeichnet den Baum auf die Grafikoberfläche (wird im Hintergrund aufgerufen). Die Position und der Maßstab des Baumes werden basierend auf der ID berechnet:

    • links, Mitte oder rechts

      Ob der Baum links, in der Mitte oder rechts gezeichnet wird, hängt von dessen ID (this.id) ab, mithilfe derer der Positionsindex POS_INDEX bestimmt wird.

      final int POS_INDEX = this.id % 3;

      Für die folgenden Berechnungen nehmen wir an, dass 0=links, 1=Mitte und 2=rechts ist. Da nur drei Positionsindizes zur Verfügung stehen, wird bei der Berechnung der Modulooperator % verwendet. Da hier immer der Rest einer Division durch 3 als Positionsindex verwendet wird, liegen die Indizes zwischen 0 und 2, obwohl die IDs bei 1 beginnen.

    • Berechnung der Position:

      Die X-Position wird anhand des Positionsindex im vorhandenen Bereich zur Platzierung von Bäumen (zwischen X_MIN und X_MAX) bestimmt. Da drei Positionsindizes möglich sind, ergeben sich drei Positionen im Bereich von 150 bis 650, wo ein Baum platziert werden kann.

      final int x = (int)(POS_INDEX / 2.0 * (MAX_X - MIN_X) + MIN_X);

      Die Y-Position wird ebenfalls durch den Index bestimmt, wodurch die Bäume in verschiedenen Höhen gezeichnet werden.

      final int y = 430 - Math.abs(POS_INDEX - 1) * 20;

      Math.abs(POS_INDEX - 1) ist 0 für Bäume in der Mitte (POS_INDEX=1) und in allen anderen Fällen dank des Betrages (Math.abs) 1. D.h. der Baum in der Mitte wird an der y-Position 430 gezeichnet, während die beiden seitlichen Bäume an 430-20 = 410 dargestellt werden

    • Berechnung des Maßstabs:

      Der Maßstab des Baumes variiert ebenfalls je nach Index, wodurch die Bäume in verschiedenen Größen gezeichnet werden.

      final double scale = 1.25 - Math.abs(POS_INDEX - 1) * 0.5;

      Auch hier ist Math.abs(POS_INDEX - 1) 0 für Bäume in der Mitte (POS_INDEX=1) und 1 für Bäume links oder rechts. Der Skalierungsfaktor ist also entweder 1.25 oder 0.75.

    • Bestimmung des Bildes:

      Das zu zeichnende Bild wird basierend auf der Jahreszeit des Baumes ausgewählt. Zum Vergleich von Strings wird stets die Instanzmethode equals der Klasse String verwendet. Diese wird auf dem Instanzattribut jahreszeit aufgerufen und dessen Inhalt jeweils mit dem übergebenen String verglichen. Sofern kein passender String gefunden wurde, wird das "POW"-Bild angezeigt, d.h. es wurde eine ungültige Jahreszeit im Attribut abgelegt. Auf die einzelnen Bilder kann mit Konstanten der Klasse TreeLibrary zugegriffen werden. 

      final Image img;
if (this.jahreszeit.equals("herbst")) {
    img = TreeLibrary.AUTUMN;
} else if (this.jahreszeit.equals("winter")) {
    img = TreeLibrary.WINTER;
} else if (this.jahreszeit.equals("frühling")) {
    img = TreeLibrary.SPRING;
} else if (this.jahreszeit.equals("sommer")) {
    img = TreeLibrary.SUMMER;
} else if (this.jahreszeit.equals("spätherbst")) {
    img = TreeLibrary.LATE_AUTUMN;
} else {
    img = TreeLibrary.POW;
}
    • Zeichnen des Bildes:

      Das ausgewählte Bild wird an der berechneten Position mit dem berechneten Maßstab gezeichnet. Dazu wird die Instanzmethode drawImage auf dem Zeichenkontext g (Übergabeparameter der Methode render) aufgerufen. Die Variable img referenziert zu diesem Zeitpunkt das für die Jahreszeit passende Bild.

      g.drawImage(img, x, y, scale);
